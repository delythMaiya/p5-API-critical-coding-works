Reflecting on the reading, I understand an API to be a bridge between different softwares, enabling cross-platform data usage.

I was initially confused about how the Google Maps API is used by companies, but I further discovered that Airbnb uses it to display business hours of nearby shops alongside rental locations. Delivery couriers like FedEx utilize the coordinate systems to link with a driver’s GPS, and many cafes use a "Store Locator" to place each location on an interactive map. I notice this especially when the interface allows me to jump directly to the Google Maps app.

I really like the quote by Burnham: "The significant artist strives to reduce the technical and psychical distance between the artistic output and the productive means of society." By using APIs that are widely available, real-time, and relevant to us, an artist pulls the artwork away from an elitist world where it is observed as a separate entity from afar. It becomes a visualization of the state of the world, making data accessible to a wider audience—especially when secretive data (e.g., drone data) is involved.

While I really like the idea of names being attached to every count of a drone attack death, the names in the example were taken from a feed with no direct connection, making them feel arbitrary. Since it is impossible to know exactly who died in real time, I would perhaps choose to use an API of babies born in that country. Even with a slight lag, using names of newborns added every day might be a bit more relevant than unrelated person's feed. 

Protocol   

From computer science exams before university, I was familiar with packet switching and how routers calculate the most efficient path for each packet to travel, but the analogy used in the text was refreshing and I thought of how similar it is to how living organisms act act, make decisions and move. It was also fascinating to discover that prior to DNS serves, there was a single text file HOSTS.txt to store all the conversions from name to IP address. 
When observing the inverted tree diagram, I was confused as I it signifies structure that is not decentralised. Now I understand a bit more as an entity where it has a hierarchical structure for indexing/organisation, and avoids everyone accessing through same hub, instead spread out globally and flexible. 
Also, when it mentioned how it defeats the purpose to imagine the meanings behind protocols, I totally agreee but I also thought how can you even extract meanings from it? Maybe someone could look for a political meaning if a protocol does not allow certain operations or commands. Looking further into this, I found an exmaple that made me understand this point more. For exmaple, a dating app allows a user to swipe left or right to get a match, and the agreed protocol between the user and the device is this motion of fingers, not through in-built gyroscope sensors or camera sensors. To make a political point would be to claim that this protocol excludes certain demographic who are unable to use their fingers, when this point should be aimed for design, not the protocol.
